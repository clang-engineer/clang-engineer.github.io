# develop style

- 개발과정도 결과만큼 중요하게 생각합니다.
- 꼼꼼한 설계 후 작업을 시작합니다.
- 작업과정에서 TDD, DDD를 적극 활용합니다.
- program <-> user-interface사이에서 데이터 흐름을 관장하는 작업을 선호합니다.
- 유지보수가 쉽고 가독성이 좋은 코드를 만들고자 합니다.
