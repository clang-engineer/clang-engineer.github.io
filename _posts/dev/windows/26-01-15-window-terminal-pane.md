## 1. Windows Terminal의 화면 구조

Windows Terminal은 하나의 **Tab** 안에  
여러 개의 **Pane(분할 화면)** 를 둘 수 있다.

- Tab: 작업 단위
- Pane: 실제 터미널 화면

> 하나의 탭 안에서 여러 작업을 병렬로 처리할 수 있는 것이 가장 큰 장점이다.

---

## 2. 화면 분할 (Pane Split)

### 2.1 단축키로 분할

#### 🔹 세로 분할
```

Alt + Shift + =

```

#### 🔹 가로 분할
```

Alt + Shift + -

```

#### 🔹 현재 포커스 기준 자동 분할
```

Alt + Shift + D

```

---

### 2.2 마우스로 분할

1. 탭 옆 `▼` 버튼 클릭
2. **Split pane** 선택
3. 실행할 터미널 선택

---

## 3. 분할된 Pane 이동

```

Alt + 방향키

```

- 키보드만으로 빠르게 pane 전환 가능
- 마우스 이동 최소화 가능

---

## 4. 분할 창 확대 (Zoom Pane)

작업 중 특정 pane에 집중하고 싶을 때  
**현재 pane만 전체 화면처럼 확대**할 수 있다.

---

### 4.1 기본 단축키가 없는 이유

다음과 같은 이유로 확대 단축키가 기본 설정에 없는 경우가 많다.

- Windows Terminal 버전 차이
- 입력기(IME) 단축키 충돌
- 사용자 정의 키맵 우선 적용

👉 따라서 **직접 단축키 설정을 권장**한다.

---

### 4.2 Zoom Pane 단축키 설정 방법

#### ① 설정 열기
```

Ctrl + ,

```
또는  
탭 옆 `▼` → **Settings**

---

#### ② Actions → Add new

- **Action**
```

Toggle pane zoom

```

- **Keys (추천 예시)**
```

Ctrl + Shift + Z

```
또는
```

Alt + Enter

```

설정 저장

---

### 4.3 사용 방법

- 확대할 pane에 포커스
- 설정한 단축키 입력
- 현재 pane만 확대
- 다시 입력하면 원래 분할 상태로 복귀

---

## 5. Pane 닫기

```

Ctrl + Shift + W

```

- 현재 포커스된 pane만 닫힘
- 탭 전체 종료와 다름

---

## 6. 자주 쓰는 화면 관련 단축키 정리

| 기능 | 단축키 |
|----|----|
| 분할 | Alt + Shift + D |
| 이동 | Alt + 방향키 |
| 확대 | Ctrl + Shift + Z |
| 닫기 | Ctrl + Shift + W |

---

## 7. 화면 기능 활용 예시

- 로그 모니터링 + 명령 실행 동시 작업
- 서버 상태 확인 + 배포 명령 분리
- Git 작업 + 빌드 출력 병렬 확인

👉 **탭은 컨텍스트 분리**,  
👉 **Pane은 작업 병렬화** 용도로 쓰면 효율이 극대화된다.

---

## 8. 정리

- Windows Terminal은 탭 + pane 구조를 기반으로 한다
- 화면 분할과 확대 기능만 익혀도 작업 속도가 크게 향상된다
- Zoom 기능은 직접 단축키를 설정하는 것이 가장 안정적이다

---
